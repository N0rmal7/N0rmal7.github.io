有关glibc的一些基础知识

前言

在做pwn题时总会遇到一些有关glibc的问题，于是重新学习了一下glibc，总结了一些知识点以及在做题时遇到glibc版本问题的解决方法。

glibc概要

glibc(GNU C Library)是 Linux 系统中面向 C 语言程序的标准库实现,其为C语言程序提供了必要的函数和工具，以处理文件操作、内存管理、字符串处理、数学运算等常见的任务。它封装了应用程序与内核交互的底层细节，为程序员提供了丰富的函数集合，从而能够专注实现应用程序逻辑，且不需要去深入了解操作系统内核的具体细节。
Linux C标准库简介

初学pwn的时候可以这么理解glibc：它本质上是一个“工具箱”，将一些最基础的功能（如输入输出、内存分配、字符串处理等）封装整合成了函数，通过调用这些函数，程序员可以轻松地实现各种功能，而不需要从头编写基础功能代码，从而省去了大量的开发时间和精力。

同时，为了修复漏洞或者增加新功能，glibc库这一“工具箱”不断更新，其中“工具”的“类型与尺寸”也会发生改变——即不同glibc版本下，一些函数的结构、功能和地址偏移量也随之不同。

做题时遇到的实际问题与解决方法

在做pwn题时，不同glibc版本的特性对于找出并解决题目漏洞起到重要作用。有些题目往往会给出题目附件之外的 ‘libc.so.6’ 文件，这往往代表题目远端环境使用的libc库与本地环境使用的可能不同(这导致libc地址偏移量的不同)，需要我们手动绑定来达到与远端环境所使用的libc库一致的效果。

笔者在初学时ret2libc时疑惑为什么在本地打通后换到远端就不行了，本质就是因为libc库调用的版本不同，函数地址在libc中的固定偏移量不同，导致一开始泄露的got表地址就是错误的。
为了解决这个问题，需要用到 patchelf 这一工具来修改题目文件所绑定的libc.so.6文件(以及动态链接器文件ld-xxx.so)，使其在本地调试时的数据与远端环境的libc版本一致，从而避免问题发生。

patchelf使用指令
patchelf --set-interpreter ./ld文件路径 ./程序文件名 # 修改程序的动态链接器路径
patchelf --set-rpath ./libc.so.6文件所在目录 ./程序文件名 # 修改程序文件绑定的libc.so.6文件路径，直接搜所在目录能自动寻址
